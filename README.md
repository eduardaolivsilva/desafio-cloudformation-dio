# desafio-cloudformation-dio
🧠 Desafio: Infraestrutura Automatizada com AWS CloudFormation
📋 Descrição

Este projeto faz parte da formação da DIO e tem como objetivo implementar uma infraestrutura automatizada utilizando AWS CloudFormation, consolidando conhecimentos sobre infraestrutura como código (IaC) e boas práticas de automação na nuvem.

🚀 Objetivos de Aprendizagem

Aplicar conceitos de CloudFormation em um ambiente prático.

Criar e gerenciar recursos AWS por meio de templates YAML.

Documentar processos técnicos de forma clara e organizada.

Publicar documentação técnica no GitHub.

⚙️ Tecnologias Utilizadas

AWS CloudFormation

AWS Management Console / AWS CLI

Git e GitHub

Markdown (.md)

🏗️ Estrutura do Projeto

template.yaml: Template principal com os recursos da AWS.

/images: Capturas de tela da execução.

README.md: Documentação detalhada do processo.

notes.md: Observações e comandos utilizados durante a prática.

📂 Recursos Criados (exemplo)

VPC, Subnets e Internet Gateway

EC2 Instance

S3 Bucket

Security Groups

Outputs com IPs e URLs

💡 Aprendizados e Insights

Automação de infraestrutura com IaC aumenta consistência e reduz erros manuais.

Uso de parâmetros e outputs facilita o reuso do template.

Boas práticas de versionamento no GitHub ajudam na rastreabilidade e colaboração.
